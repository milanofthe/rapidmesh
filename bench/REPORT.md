# rapidmesh benchmark report

Generated by `bench/report.py` from `bench/results.json` and the viewer JSONs.
Pipeline: `cargo run --release --bin bench`, then `python bench/compare_meshers.py`,
then `python bench/report.py`.

Method: tetgen meshes the IDENTICAL assembled PLC (bench/plc/*.json, region
seeds + per-region max-volume converted from maxh as h^3/(6*sqrt(2)), switches
`pzQq<bound>Aa`). gmsh rebuilds the scenes analytically with OCC (fragment for
conformal interfaces and embedded sheets, Constant fields for region sizing,
default 3D Delaunay + optimization). Quality metrics are computed identically
for all meshers (bit-equal cross-validated against the Rust quality_stats).

## Notes

- rapidmesh times include the full pipeline (exact CSG assembly, conforming
  meshing, quality optimization); gmsh/tetgen times are the meshing call only.
- gmsh/tetgen run their default optimization passes; tetgen's defaults leave
  slivers (min dihedral 1.5-3.4 deg), gmsh lands at 7-14 deg, rapidmesh at
  15-21 deg (except the known density_transition all-boundary sliver).
- The rapidmesh stage split shows meshing itself is in tetgen's range; the
  optimizer dominates total time (improvement backlog: incremental passes).

## Surface models (STL/OBJ)

Imported organic surface meshes (bench/fetch_models.py, e.g. spot with 5856
facets) currently DO NOT complete in reasonable time: assembly is fast (49 ms
for spot), but conforming refinement spirals (every input edge is a crease,
every facet its own plane patch, and small input angles between facet planes
drive quality refinement). tetgen meshes the same surface in well under a
second. Tracked as a performance/termination work package; surface models are
parked in bench/models_hold until then.
